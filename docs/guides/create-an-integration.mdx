## Authentication and credential management

superglue supports multiple authentication methods with automatic token management and refresh. All credentials are encrypted at rest and in transit.

<AccordionGroup>
  <Accordion title="OAuth" icon="user-lock">
    OAuth is used for secure delegated access. superglue supports both client credentials flow and authorization code flow with **automatic token refresh**.

    **What superglue handles automatically:**
    - OAuth flow management
    - Scope validation
    - Error handling for token issues
    - Token exchange and storage
    - Token refresh when expired

    ### Authorization code flow
    For user-facing applications where users grant permission to access their data:

    1. User clicks authorize button
    2. Redirected to provider's login page
    3. User grants permission with requested scopes
    4. superglue receives authorization code and exchanges it for access token
    5. Tokens are automatically refreshed when they expire

    **Examples:** HubSpot, Salesforce, Google Workspace

    ### Client credentials flow
    For server-to-server communication with no user interaction:

    1. Application requests access token using client ID and secret
    2. Token is automatically refreshed before expiration
    3. No user authorization required

    **Examples:** Twilio, Auth0 Management API, Microsoft Graph

    ### Configuration options
    For custom OAuth integrations, provide all required fields:

    ```typescript
    await superglue.upsertIntegration({
        id: "custom-oauth",
        urlHost: "https://api.example.com",
        documentationUrl: "https://docs.example.com/quickstart"
        credentials: {
            clientId: "your_client_id",
            clientSecret: "your_client_secret",
            authorizationUrl: "https://auth.example.com/oauth/authorize",
            tokenUrl: "https://auth.example.com/oauth/token",
            scopes: "read:users write:data",
        }
    });
    ```

    **Required fields:**
    - `clientId`: Your OAuth application's client ID
    - `clientSecret`: Your OAuth application's client secret
    - `authorizationUrl`: Where users are redirected to authorize
    - `tokenUrl`: Where superglue exchanges codes for tokens

    **Optional fields:**
    - `scopes`: String of space seperated OAuth scopes (e.g., `"read:users write:data`)
    - `redirectUri`: Where users are redirected after authorization


    <Info>
      We have pre-built OAuth templates for 100+ popular APIs including HubSpot, Google Ads, Salesforce, and more. Check the templates dropdown when creating an integration to see what's available.
    </Info>
  </Accordion>

  <Accordion title="API keys" icon="key">
    The simplest authentication method. Just provide your API key and superglue figures out where to place it based on the API documentation.

    ```typescript
    credentials: {
      apiKey: "your_api_key_here"
    }
    ```

    superglue automatically determines the correct placement:
    - Authorization header: `Authorization: Bearer {key}`
    - Custom header: `X-API-Key: {key}`
    - Query parameter: `?api_key={key}`

    ### Examples

    <Tabs>
      <Tab title="SendGrid">
        ```typescript
        await superglue.upsertIntegration({
          id: "sendgrid",
          urlHost: "https://api.sendgrid.com",
          documentationUrl: "https://docs.sendgrid.com/api-reference",
          credentials: {
            apiKey: "SG.xxx..."
          }
        });
        ```
        
        SendGrid requires the API key in a custom Authorization header with Bearer prefix.
      </Tab>
      <Tab title="Airtable">
        ```typescript
        await superglue.upsertIntegration({
          id: "airtable",
          urlHost: "https://api.airtable.com",
          documentationUrl: "https://airtable.com/developers/web/api/introduction",
          credentials: {
            apiKey: "pat..." // Personal access token
          }
        });
        ```
        
        Airtable uses a personal access token passed as a Bearer token in the Authorization header.
      </Tab>
      <Tab title="Zendesk">
        ```typescript
        await superglue.upsertIntegration({
          id: "zendesk",
          urlHost: "https://yourcompany.zendesk.com",

          credentials: {
            email: "admin@yourcompany.com",
        ```

        Zendesk requires both email and API token, combined as basic auth: `{email}/token:{apiKey}`.

      <Tab title="Shopify">

## Automatic documentation fetching

One of superglue's key features is the handling the information gathering process per integration, so you don't have to manually provide API specifications.

### Supported documentation formats

superglue automatically fetches and processes documentation from various sources:

<CardGroup cols={2}>
  <Card title="OpenAPI/Swagger" icon="file-code">
    Automatically parsed for endpoints, parameters, and schemas. Special handling for OpenAPI 3.0+ specifications.
  </Card>
  <Card title="Mintlify & structured docs" icon="book-open">
    Extracted from modern documentation sites with structured content.
  </Card>
  <Card title="GraphQL introspection" icon="diagram-project">
    Automatic schema discovery via introspection queries. No manual configuration needed.
  </Card>
  <Card title="Database schemas" icon="database">
    PostgreSQL and MySQL schema analysis for SQL integrations. Automatically discovers tables, columns, and relationships.
  </Card>
</CardGroup>

### Documentation optimization with keywords

API documentation can be very long (some have thousands of endpoints). When you already know what you want to do, superglue can fetch more specific parts of the documentation for better performance and accuracy.

<Tabs>
  <Tab title="Via Chat/Agent">
    Tell superglue what you're trying to do, and it will prioritize relevant documentation:

    ```
    "Connect to Stripe and focus on customer and subscription endpoints"
    ```

    The agent will fetch and rank documentation sections related to customers and subscriptions higher, making workflow building faster and more accurate.
  </Tab>
  <Tab title="Via SDK">
    Provide specific keywords when creating or updating an integration:

    ```typescript
    const integration = await superglue.upsertIntegration({
      id: "stripe",
      urlHost: "https://api.stripe.com",
      documentationUrl: "https://docs.stripe.com/api",
      specificInstructions: "Focus on customers, subscriptions, and invoices. Ignore payment methods and disputes.",
      credentials: {
        apiKey: "sk_test_..."
      }
    });
    ```

    Keywords and instructions will be used to:
    - Fetch more relevant documentation sections
    - Rank endpoints and schemas by relevance
    - Improve workflow building accuracy
  </Tab>
</Tabs>

**How it works behind the scenes:**
1. superglue fetches the API documentation
2. If keywords are provided, relevant sections are identified and ranked
3. Documentation is chunked and embedded for semantic search
4. When building workflows, the most relevant documentation is retrieved first
